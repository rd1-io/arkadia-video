## Сетевой сценарий 1 — Использование оптики провайдера

### Описание подключения

Каждый шкаф ВРУ уже имеет рядом колодец с оптической муфтой провайдера. PoE-коммутатор в шкафу подключается к оптической сети провайдера через SFP-медиаконвертер (или SFP-порт коммутатора напрямую, если провайдер предоставляет волокно).

```
Камеры (PoE UTP) ──→ Коммутатор Dahua PFS4210 ──→ SFP-порт ──→ Патч-корд OM3
                                                                      │
                                                               Муфта провайдера
                                                                      │
                                                         Оптическая сеть провайдера
                                                                      │
                                                          Серверная провайдера
                                                                      │
                                                           NVR / DSS Pro
                                                                      │
                                              VPN-туннель ──→ Охрана КПП (ПК/монитор)
```

### Требования к провайдеру

| Параметр | Требование |
|---|---|
| Предоставляемая полоса | не менее 1 Гбит/с на весь посёлок |
| Задержка (latency) | не более 5 мс внутри посёлка |
| VLAN для видеонаблюдения | выделенный, изолированный от жилых абонентов |
| Место для NVR | стойко-место в серверной провайдера (U2–4) |
| Доступ к NVR | 24/7, возможность физического обслуживания |
| Аварийное восстановление | SLA: не более 4 часов на устранение отказа канала |
| Запрет доступа камер в интернет | изоляция на уровне ACL/VLAN |

## Сетевой сценарий 2 — Собственная ВОЛС

### Описание

Прокладка собственного волоконно-оптического кабеля от административного здания кольцом через все узлы. Кольцо обеспечивает резервирование (ERPS G.8032 или RSTP).

```
          Адм. здание
        ┌──────────────┐
        │   CSS4024    │◄─────────────────────────────────┐
        └──────┬───────┘                                  │
               │ волокно 1-2                              │ волокно 5-6 (резерв)
        ┌──────▼───────┐                          ┌───────▼──────┐
        │   Узел 1     │                          │   Узел 8     │
        │   КПП        │                          │   Насосная   │
        └──────┬───────┘                          └───────┬──────┘
               │                                          │
        ┌──────▼───────┐                          ┌───────▼──────┐
        │   Узел 2     │                          │   Узел 7     │
        │   Пер. №1    │                          │   Пер. №6    │
        └──────┬───────┘                          └───────┬──────┘
               │                                          │
        ┌──────▼───────┐                          ┌───────▼──────┐
        │   Узел 3     │──────────────────────────│   Узел 6     │
        │   Пер. №2    │                          │   Пер. №5    │
        └──────┬───────┘                          └───────┬──────┘
               │                                          │
        ┌──────▼───────────────────────────────────▼──────┐
        │             Узлы 4, 5 (Пер. №3, №4)             │
        └──────────────────────────────────────────────────┘
```

### Кабель ВОЛС

| Параметр | Значение |
|---|---|
| Тип кабеля | ОКЛСТ-0,22-8Е1 (8 волокон, бронированный, подземный) |
| Протяжённость кольца | ~4–4,5 км (с запасом 15%) |
| Монтаж | В существующей кабельной канализации или прокладка в грунт на глубину 0,8 м |
| Пересечения с дорогами | ГНБ (горизонтальное направленное бурение), ~6 пересечений |
| Муфты | МТОК-96А в каждом узле, 8 волокон, термоусадка |
| Патч-панели | В каждом шкафу SC/APC адаптеры |
| Запас волокна | Не используемые волокна законсервировать для расширения |

### SFP-модули

| Коммутатор | SFP-модуль | Длина волны / дальность |
|---|---|---|
| Dahua PFS4210 (узлы) | SFP-1000BASE-LX | 1310 нм / 10 км |
| Dahua CSS4024 (центр) | SFP+-10G-LR или SFP-1000BASE-LX | 1310 нм / 10 км |

## Резервирование питания (ИБП)

| Узел | ИБП | Расчётная нагрузка | Время автономии |
|---|---|---|---|
| Административное здание | APC Smart-UPS 1500 ВА | NVR ~120Вт + коммутатор ~30Вт | ~6 ч |
| Каждый шкаф ВРУ (×7–8) | Аккумулятор 12В/17Ач в шкаф с зарядным | Коммутатор ~15Вт + камеры суммарно 15–30Вт | ~4 ч |

> Для уличных шкафов рекомендуется использовать ИБП в виде свинцово-кислотного аккумулятора 12В/17Ач с зарядным модулем PS-BOX или аналогичным, установленным в термошкаф с обогревом и терморегулятором.

## Безопасность сети

- Камеры и NVR — в изолированном VLAN, без доступа в интернет
- Удалённый доступ к NVR — только через VPN (WireGuard или OpenVPN на шлюзе)
- Смена заводских паролей на всём оборудовании при пусконаладке
- Отключить все неиспользуемые порты коммутаторов
- Включить port security / MAC-address binding на PoE-портах
- Логирование доступа (syslog на NVR или отдельный сервер)

## Расчёт пропускной способности

| Параметр | Вариант A | Вариант B |
|---|---|---|
| Кол-во камер | 20 | 28 |
| Реалистичный средний битрейт (8 МП ColorVu, H.265) | ~5 Мбит/с × 20 = **100 Мбит/с** | ~5 Мбит/с × 28 = **140 Мбит/с** |
| Пиковый (все камеры × 1,5) | ~150 Мбит/с | ~210 Мбит/с |
| Магистраль оптика | 1 Гбит/с — запас ~5× | 1 Гбит/с — запас ~4× |
| Узел КПП (макс. 5 камер) | 5 × 5 = 25 Мбит/с — в норме | — |
| Объём хранения 30 дней | **32,4 ТБ** (6 HDD RAID-5 = 40 ТБ) | **45,4 ТБ** (8 HDD RAID-5 = 56 ТБ) |
